# Onyx NetSurf core libraries (brick 7)

The portable library stack the **NetSurf** browser is built on, cross-compiled for Onyx
(`aarch64-none-elf` + newlib). Same split as `user/tls/` and `user/img/`: the upstream
sources are cross-built here (placed as `./<lib>` — git clones / submodules, **not**
committed), and the `Makefile` compiles each library's translation units directly,
bypassing NetSurf's own buildsystem.

The **whole stack links clean** against the Onyx newlib environment (`crt0libc.S` +
`libc/onyx_syscalls.c` + newlib + libm) with **no undefined symbols** — verified by
whole-archive linking all eight `.a` together.

## Libraries (build order = dependency order)

| library | role | deps | codegen |
|---|---|---|---|
| libwapcaplet | interned strings | — | — |
| libparserutils | parsing / charset codecs | — | perl (charset aliases) |
| libnsutils | misc utilities | — | — |
| libnsgif | GIF decoder | — | — |
| libnsbmp | BMP/ICO decoder | — | — |
| libhubbub | HTML5 parser | libparserutils | perl (entities) + element-type table |
| libcss | CSS parser + selection | libwapcaplet, libparserutils | host `gen_parser` |
| libdom | DOM | libwapcaplet, libparserutils, libhubbub | — |

`libnslog` is **not** built here (it needs flex/bison for its filter-config parser and is
not on the rendering path).

## Code generation — without the full toolchain

The upstream Makefiles run host tools to generate sources. We reproduce just what's needed,
working around the two host tools this environment lacks (`gperf`, and no passwordless apt):

- **libparserutils** `src/charset/aliases.inc` ← `perl build/make-aliases.pl`.
- **libhubbub** `src/tokeniser/entities.inc` ← `perl build/make-entities.pl`; and
  `src/treebuilder/autogenerated-element-type.c` is normally a **gperf** perfect-hash —
  gperf is unavailable, so the Makefile drops in
  [`gen/hubbub-autogenerated-element-type.c`](gen/hubbub-autogenerated-element-type.c), a
  plain-C table with the same interface (case-insensitive linear scan; the table mirrors
  `element-type.gperf` 1:1). Keeps libhubbub unpatched.
- **libcss** `src/parse/properties/autogenerated_*.c` ← a **host** tool `gen_parser`
  compiled from `css_property_parser_gen.c` with `$(BUILD_CC)`. Note: the toolchain's plain
  `cc`/`gcc` alias the *aarch64 cross* compiler, so `BUILD_CC` must name the hosted x86
  compiler (default `x86_64-linux-gnu-gcc`).

## Onyx-specific build fixes (in the Makefile / syscall layer)

- **`-DWITHOUT_ICONV_FILTER`** (libparserutils): newlib has no `iconv`, so the input charset
  filter is built to use libparserutils' **own** codecs (utf8/utf16/8859/ext8/ascii) instead.
  Covers the common web encodings; drops the `iconv*` link dependency.
- **`-fcommon`** (all): this code predates GCC 10's `-fno-common` default and relies on
  tentative globals being merged (e.g. libcss's `_ALIGNED`); without it the generated
  property parsers multiply-define it.
- **`pread`/`pwrite`**: added to `user/libc/onyx_syscalls.c` (libnsutils uses them); done as
  save-offset / seek / read|write / restore-offset over the existing fd layer.

## Building

1. Clone the sources under this directory, pinned to the tested commits:

   ```sh
   base=git://git.netsurf-browser.org
   for l in libwapcaplet libparserutils libnsutils libnsgif libnsbmp libhubbub libcss libdom; do
     git clone "$base/$l.git" user/netsurf/$l
   done
   ```

   Tested commits: libwapcaplet `c7c128d`, libparserutils `6b0cbf0`, libnsutils `0bd3906`,
   libnsgif `24ddb64`, libnsbmp `ea063c9`, libhubbub `6651b8c`, libcss `104d87f`,
   libdom `f69781e`.

2. Build (needs perl + a hosted x86 compiler for `BUILD_CC`):

   ```sh
   make -C user/netsurf
   ```

   → `user/netsurf/<lib>/<lib>.a` for each library.

## Fetcher (brick 8)

[`onyx_fetch.c`](onyx_fetch.c) is an Onyx **fetch scheme handler** for NetSurf — the
equivalent of `content/fetchers/curl.c` but driving the Onyx TCP kapis instead of libcurl.
It implements `struct fetcher_operation_table` (modelled on the simple `data:` fetcher):
`setup()` rings a context, `poll()` runs each fetch to completion and delivers
`FETCH_HEADER` / `FETCH_DATA` / `FETCH_FINISHED` (or `FETCH_ERROR`) via
`fetch_send_callback()`. Transport is a blocking HTTP/1.0 GET over
`kapi_tcp_connect/send/recv/close` into a growable buffer, with **gzip/deflate** decoding
via zlib. Registered for the `http` scheme by `fetch_onyx_register()`.

It **compiles clean against the real NetSurf fetcher API** (verified with `-fsyntax-only`
against `content/fetch.h` + `content/fetchers.h` + zlib + the kapi). First cut: `http://`
+ GET + gzip; `https://` registers separately once the C++ TLS transport
(`user/tls/onyx_tls.hpp`) is wired in; a non-blocking, `fdset`-driven version comes later.

### compat/ — platform shims

NetSurf's `utils/inet.h` (pulled into every core TU via `content/fetch.h`) needs the BSD
sockets headers. newlib ships `<sys/select.h>` (fd_set) but not `<sys/socket.h>` /
`<netinet/in.h>` / `<arpa/inet.h>`, so [`compat/`](compat/) provides minimal versions (just
the types NetSurf references — Onyx does TCP through the kapi, not BSD sockets). These are
the start of the platform layer the full core build (brick 9) needs.

## Core + frontend (brick 9) — **it links**

NetSurf's **framebuffer frontend** (it already drives libnsfb) builds on the Onyx `"onyx"`
libnsfb surface (`user/nsfb`), with the internal bitmap font (no freetype), our `onyx_fetch`
fetcher, and the `user/img` decoders — bypassing NetSurf's buildsystem and compiling the TUs
directly (as for the libs). The frontend picks its surface **by name** (`nsfb_type_from_name`),
so no NetSurf code change is needed — just select `onyx`.

**The whole browser links.** `make -f netsurf-app.mk` compiles **182 objects** (core +
handlers + framebuffer frontend + Onyx glue) and links them with the eight libraries +
`libnsfb` (onyx surface) + the codecs + newlib into a **~4.4 MB AArch64 `netsurf.elf`** —
zero undefined symbols, with `_start` (crt0libc) and the frontend `main`.

Codegen done at build time (host tools): the internal bitmap font (`convert_font` on
`res/fonts/glyph_data`) and the toolbar/pointer/throbber bitmaps (`convert_image` on the res
PNGs — needs host libpng). Excluded: `content/fetchers/curl.c` (replaced by `onyx_fetch`),
`font_freetype.c`, and the optional image formats (svg/webp/jxl/rosprite/gst).

**Platform layer** (`compat/` + `gen/`) — the newlib/OS gaps the buildsystem normally fills:

- `compat/sys/socket.h`, `netinet/in.h`, `arpa/inet.h` — BSD sockets types (`utils/inet.h` is
  pulled into every core TU via `content/fetch.h`); `compat/sys/mman.h` — `mmap` (file:// maps
  files, implemented as malloc+pread); `compat/curl/curl.h` — stub (curl is `#include`d but not
  registered); `compat/dirent.h` + `compat/sys/utsname.h`.
- `compat/onyx_compat.c` — implementations: `uname`, directory stubs, `mmap`/`munmap`, a
  **passthrough `iconv`** (correct for the UTF-8 path; full conversion later), **stub POSIX
  regex** (used only by non-core features), `inet_aton`/`pton`/`ntop`, and `stat`/`access`/
  `mkdir`/`ftruncate`/`realpath`/`scandir`/`socket`/… shims.
- `compat/onyx_nsconfig.h` — force-included build-config macros (`-Dnsframebuffer`,
  `NETSURF_BUILTIN_*`, `PATH_MAX`, `NETSURF_HOMEPAGE`, `GECOS`/`USERNAME`, fb paths).
- `gen/testament.h` — the version stamp NetSurf generates from git.
- libdom's **hubbub HTML binding** (`bindings/hubbub/parser.c`) is built into `libdom.a` (the
  `user/netsurf` lib Makefile) — NetSurf drives HTML parsing through `dom_hubbub_parser_*`.

**Remaining for a *running* browser** (not yet done):

- **Resources at runtime**: NetSurf loads a `Messages` catalogue, a default CSS, and the
  about: pages from its resource path — these must be staged onto the SD card and the resource
  path pointed at them (the buildsystem normally embeds `resources/`).
- **Run it on hardware**: a `main()` loop is present (the fb frontend); first light is a local
  `file://` page on the onyx surface. Directory listing, real fonts beyond the internal one,
  and the `iconv`/regex shims are first-cut.
- **`https://`** via the C++ TLS transport (a second fetcher registration), then **brick 10**:
  secure TLS (CA bundle + cert verification).
