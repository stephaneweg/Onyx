# Onyx NetSurf core libraries (brick 7)

The portable library stack the **NetSurf** browser is built on, cross-compiled for Onyx
(`aarch64-none-elf` + newlib). Same split as `user/tls/` and `user/img/`: the upstream
sources are cross-built here (placed as `./<lib>` — git clones / submodules, **not**
committed), and the `Makefile` compiles each library's translation units directly,
bypassing NetSurf's own buildsystem.

The **whole stack links clean** against the Onyx newlib environment (`crt0libc.S` +
`libc/onyx_syscalls.c` + newlib + libm) with **no undefined symbols** — verified by
whole-archive linking all eight `.a` together.

## Libraries (build order = dependency order)

| library | role | deps | codegen |
|---|---|---|---|
| libwapcaplet | interned strings | — | — |
| libparserutils | parsing / charset codecs | — | perl (charset aliases) |
| libnsutils | misc utilities | — | — |
| libnsgif | GIF decoder | — | — |
| libnsbmp | BMP/ICO decoder | — | — |
| libhubbub | HTML5 parser | libparserutils | perl (entities) + element-type table |
| libcss | CSS parser + selection | libwapcaplet, libparserutils | host `gen_parser` |
| libdom | DOM | libwapcaplet, libparserutils, libhubbub | — |

`libnslog` is **not** built here (it needs flex/bison for its filter-config parser and is
not on the rendering path).

## Code generation — without the full toolchain

The upstream Makefiles run host tools to generate sources. We reproduce just what's needed,
working around the two host tools this environment lacks (`gperf`, and no passwordless apt):

- **libparserutils** `src/charset/aliases.inc` ← `perl build/make-aliases.pl`.
- **libhubbub** `src/tokeniser/entities.inc` ← `perl build/make-entities.pl`; and
  `src/treebuilder/autogenerated-element-type.c` is normally a **gperf** perfect-hash —
  gperf is unavailable, so the Makefile drops in
  [`gen/hubbub-autogenerated-element-type.c`](gen/hubbub-autogenerated-element-type.c), a
  plain-C table with the same interface (case-insensitive linear scan; the table mirrors
  `element-type.gperf` 1:1). Keeps libhubbub unpatched.
- **libcss** `src/parse/properties/autogenerated_*.c` ← a **host** tool `gen_parser`
  compiled from `css_property_parser_gen.c` with `$(BUILD_CC)`. Note: the toolchain's plain
  `cc`/`gcc` alias the *aarch64 cross* compiler, so `BUILD_CC` must name the hosted x86
  compiler (default `x86_64-linux-gnu-gcc`).

## Onyx-specific build fixes (in the Makefile / syscall layer)

- **`-DWITHOUT_ICONV_FILTER`** (libparserutils): newlib has no `iconv`, so the input charset
  filter is built to use libparserutils' **own** codecs (utf8/utf16/8859/ext8/ascii) instead.
  Covers the common web encodings; drops the `iconv*` link dependency.
- **`-fcommon`** (all): this code predates GCC 10's `-fno-common` default and relies on
  tentative globals being merged (e.g. libcss's `_ALIGNED`); without it the generated
  property parsers multiply-define it.
- **`pread`/`pwrite`**: added to `user/libc/onyx_syscalls.c` (libnsutils uses them); done as
  save-offset / seek / read|write / restore-offset over the existing fd layer.

## Building

1. Clone the sources under this directory, pinned to the tested commits:

   ```sh
   base=git://git.netsurf-browser.org
   for l in libwapcaplet libparserutils libnsutils libnsgif libnsbmp libhubbub libcss libdom; do
     git clone "$base/$l.git" user/netsurf/$l
   done
   ```

   Tested commits: libwapcaplet `c7c128d`, libparserutils `6b0cbf0`, libnsutils `0bd3906`,
   libnsgif `24ddb64`, libnsbmp `ea063c9`, libhubbub `6651b8c`, libcss `104d87f`,
   libdom `f69781e`.

2. Build (needs perl + a hosted x86 compiler for `BUILD_CC`):

   ```sh
   make -C user/netsurf
   ```

   → `user/netsurf/<lib>/<lib>.a` for each library.
